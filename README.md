# TwiChatFHR (TwitchChatCore)

> [!IMPORTANT]
> Это легковесный и красивый чат для OBS со **встроенным обходом блокировок** (например, для 7TV). Он настраивается за пару кликов и будет работать безотказно — главное, чтобы был интернет и подключение к Twitch.

TwiChatFHR — это современный, настраиваемый клиент и оверлей для Twitch-чата, написанный на C# с использованием Avalonia UI и встроенного локального веб-сервера. Основная цель проекта — предоставить стримерам быстрый, расширяемый и визуально привлекательный чат, независимый от тяжелых браузерных решений.

## Зачем нужен этот проект?

Стандартные веб-клиенты Twitch-чата часто бывают перегружены, потребляют много ресурсов или не предоставляют достаточной гибкости для кастомизации под нужды конкретного стримера. TwiChatFHR решает эти проблемы путем разделения логики на производительный бекенд (C#) и легковесный, полностью настраиваемый фронтенд (HTML/JS/CSS).

Основные преимущества:
- Сниженное потребление ресурсов по сравнению с полноценным браузером.
- Полный контроль над дизайном чата через локальные веб-компоненты.
- Встроенные инструменты модерации, которые работают на стороне клиента.
- Поддержка сторонних провайдеров расширений (7TV, BTTV, FFZ) "из коробки", а также возможность обхода их блокировок.

## Ключевые возможности

- Прямое подключение к Twitch IRC: Быстрое и надежное получение сообщений в реальном времени.
- Встроенный веб-сервер Kestrel: Поднимает локальный сервер для рендеринга виджета чата, позволяя легко добавлять его в OBS.
- Обход блокировок (Proxy): Встроенная возможность использовать Google proxy для обхода региональных блокировок сторонних сервисов (например, 7TV). Пользователь может легко настроить прокси самостоятельно — подробный гайд доступен прямо в настройках приложения.
- Кэширование ресурсов: Автоматическая загрузка и кэширование значков (badges) и эмотов (emotes), что снижает сетевую нагрузку и ускоряет отображение.
- Инструменты модерации: Поддержка локальных черных списков пользователей (`blacklist.txt`) и авто-бана по ключевым фразам (`ban_phrases.txt`).
- Режим тестирования: Встроенный симулятор чата для настройки внешнего вида без необходимости запускать трансляцию или писать самому себе.
- Кроссплатформенность: Использование фреймворка Avalonia UI обеспечивает возможность работы приложения на различных операционных системах.

> [!TIP]
> Вы можете протестировать отображение виджета, введя `test` в качестве названия канала в настройках. Это запустит встроенный генератор, который будет отправлять тестовые сообщения с различными значками и эмотами.

## Архитектура проекта

Проект разделен на несколько ключевых модулей:
- Desktop UI (`Views`, `App.axaml`): Отвечает за графический интерфейс десктопного приложения (настройки, превью чата).
- Локальный сервер (`Server/LocalServerManager.cs`): Микро-сервер на базе Kestrel, обрабатывающий WebSocket-соединения и раздающий статические файлы виджета.
- Управление ресурсами (`Server/BadgeManager.cs`, `Server/EmoteManager.cs`): Интеграция с API Twitch и сторонних сервисов для загрузки графических ассетов.
- Web-фронтенд (`browser/`): Статические файлы (HTML, JS, CSS), представляющие визуальный слой чата. Общение с бекендом происходит асинхронно через WebSockets (`ChatHub.cs`).

> [!WARNING]
> При редактировании стилей или скриптов в папке `browser` нет необходимости пересобирать весь C#-проект. Достаточно обновить страницу виджета в OBS или окне превью, так как файлы отдаются локальным сервером напрямую из директории.

## Инструкция по сборке

Для работы с кодом и сборки проекта потребуется .NET 8 SDK.

1. Склонируйте репозиторий на ваш компьютер.
2. Перейдите в корневую директорию проекта.
3. Выполните восстановление зависимостей и запуск:

```bash
dotnet restore
dotnet build
dotnet run
```

> [!IMPORTANT]
> Для корректного отображения превью чата внутри десктопного приложения в системе Windows должен быть установлен компонент Microsoft Edge WebView2 Runtime.
